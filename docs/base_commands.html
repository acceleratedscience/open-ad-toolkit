The base commands enable you to navigate and work with data fed into or generated by User Toolkits. they also enable you to administor your environment

General Commands

Get Status:

get status
Returns the current workspace and toolkit context.

Display Command History:

display history
displays command history for the current workspace.

Display Data:

display data <csv_filename>

displays the data of a csv file.

Clear Sessions:

clear sessions

clear sessions is only required to be run when adding or removing a toolkit or workspace. The Toolkit allows you to run parallel sessions, if for some reason you close a terminal or you have a crash a session may be left open. In this case a Wokrspace or Toolkit cannot be added or removed until other sessions are shut down and any zombie sessions are cleared.

Only Use this command when you are sure all other sessions are not running.

Toolkits

Remove Toolkit:

remove toolkit <toolkit_name>
Removes the specified toolkit from the list of available toolkits.

Add Toolkit:

add toolkit <toolkit_name>
Adds a toolkit from the installed available toolkits. to see available toolkts see the 'list all toolkits' command.

List Toolkits:

list toolkits
lists the current added toolkits that users have access to.

List All Toolkits:

list all toolkits
lists all the available toolkits to add.

Set Context:

set context <toolkit_name>
Sets the curent toolkit that the user can have access to.

Unset context:

unset context
removes th current toolkit from context, leaving no toolkits in context.

Workspaces

Set Workspace:

set workspace <workspace_name>
Sets the current workspace the user will work in.

Create Workspace:

create workspace <workspace_name> [decription('<description for workspace>')][on path '<absolute_path>']
Creates a Workspace by default within the working directory. Optionally a workspace can be created on a separate path. If the user does not add a description from the command line or Notebook the user will be prompted for a description.

Remove Workspace:

remove workspace <workspace_name>
removes the specified workspace from the list of available workspaces. It does not delete the workspace directory, so if the workspace is added again, all files and runs will be accessible again.

Get Workspace:

get workspace <workspace_name>
lists the details of the named workspace.

List Workspaces:

list workspaces
lists all available workspaces

Runs

List Runs:

list runs
Lists all runs for the currently selected workspace

Create Run:

create run
Flags the creation of a run. Use this command to lay down the starting point for the beginging of a run. It places a marker in the command history file for the creation of a run.

Save Run:

save run as <run_name>
This command will signal the end of recording of a run. The client will then back track through the history file to find the create run flag and place all commands before it into the run file.

Display Run:

display run as <run_name>
This command displays the named run, showing each command in sequence.

Run:

run <run_name>
This command will execute a run, as each command in the run completes it will display the out put from that command.

File System commands

List Files:

list files
Lists the files in the current workspace.

Import Files:

import from '<source_file_path>' to '<destination_filename>'
this command imports a file defined with an absolute source file path to a destination file in the current workspace. For example: 
import from '/home/user1/documents/molecules.sdf' to 'source_mols.sdf'


Import Files:

import from '<destination_file_path>' to '<source_filename>'
this command imports a file defined with an absolute source file path to a destination file in the current workspace. For example: 
import from '/home/user1/documents/molecules.sdf' to 'source_mols.sdf'
