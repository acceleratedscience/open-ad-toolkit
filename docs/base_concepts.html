Open toolkit Framework
v0.01 - IBM Github

The Open toolkit Framework is designed to provide a foundation for aggregating scientific tools together into one API that can be accessed by Command Line, Jupyter Notebook Magic Commands, and a Python API. The API allows users no, matter their entry point, to define commands via a Domain Specific Language (DSL) that is common across all API interfaces. Like SQL is to Databases it is providing a simple DSL for traversing workspaces, calling functions from different services as well as working with molecular python projects such as rdkit, mols2grid, gt4sd, ST4SD and rxn for Chemistry.

The goal of the Framework is to make it easy for scientists to go about their day to day work with a common language to use end ot end in their molecular AI driven projects.

Workspaces

Workspaces are a space where you can conduct activities that grouped together and have their own runs, files and history of commands. Files can be imported and exported out of workspaces. Workspaces can be created int the default workspace directory 

~/.openad/workspaces/CAPITALISED_WORKSPACE_NAME
or their location can be defined at time of creation. the toolkit will start with a default Workspace to begin with. All operations run within a workspace. Workspace Related Commands are:
	set workspace <workspace_name>
	create workspace <workspace_name> [ description('<workspace_description>') on path '<path>' ]
	remove workspace <workspace_name>
	list workspaces
	get workspace <workspace_name>

Context

Context defines the User Toolkit that is current to be used up and above the base commands in the framework. the concept is that if a user sets the current context to RXN then the toolkit ensures there is a connection to RXN and all the latest commands for RXN are loaded for the user to use. If then the user wants to move to Deep search they need to set the context to another User Toolkit 

 set context DS4SD
for example. this would then switch the user to DeepSearch toolkit and try to log the user on to their current DeepSearch Service. If it is the first time the sets the context to that service it will request the user to enter their login details. If the user needs to change those details they can set context with the 'reset' option which will have the user re-enter their logon details. Context Related Commands are:
	set context <user_toolkit_name> | reset
	unset context

information on the base commands are available here. openad commands.

User Toolkits

A user toolkit is a toolkit that has been defined to provide a logon and commands to a Accelerated Discovery Service such as RXN for chemistry or Deep Search. The Framework Ships IBM Toolkits for RXN and Deep Search Currently with the Framework and plans to add ST4SD and GT4SD shortly. To add a toolkit such as Deepsearch you simply would enter or call the following command.

 add toolkit DS4SD
for example.
In future Users will be able to add their own Toolkits or create their own customised versions of the IBM Toolkits, and run them.
For toolkits that are not Service specific, an extension will also be added to cater for these types of toolkits, e.g. tools that display or working with molecules. Toolkit Related Commands are:
	add toolkit <user_toolkit_name> 
	remove toolkit <user_toolkit_name> 
 	list toolkits 
 	list all toolkits
