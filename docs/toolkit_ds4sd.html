Introduction

The Deep Search toolkit integration provides a means to search a Deep Search Repository from the command line or Notebook.
Deep Search leverages state-of-the-art AI methods to continuously collect, convert, enrich, and link large document collections. You can use it for both public and proprietary documents.
To access your deepsearch API key, which is necessary for use with the Opentoolkit see Deep Seearch Website .

Commands:

Display A Collection

This function enables you to display the details an available collections to query.
display collection details'<collection_name>| <collection_index>'

Example Command: display collection details 'Materials Science'

Display All Collections

This function displays all available collections in Deep Search.
If you use the SAVE AS clause, it will save a csv file to the current workspace."
display all collections

Example Command: display all collections

Display Collections in Domains

This function displays collections that belong to the listed domains.
f you use the SAVE AS clause, it will save a csv file to the current workspace.
display collections in domains from list [<list_of_domains>] [save as '<csv_file_name >']

Example Command: display collections in domains from list ['Scientific Literature','Healthcare & Life Sciences']

Display Collection Matches

This command searches all collections for documents that contain a given Deep Search <search_string>. This helps choose document collection(s) for subsequent search. Use <index_key> from the returned table in a search.
If you use the SAVE AS clause, it will save a csv file to the current workspace.
display collection matches for '<search_string>' [save as '<csv_file_name>']

Example Command: display collection matches for '\"Carbon\" AND \"Capture\"'

Display Collections For Domain

This command displays the available collections in a given Deep Search <domain_name>.
If you use the SAVE AS clause, it will save a csv file to the current workspace.
display collections for '<domain_name>' [save as '<csv_file_name>']

Example Command: display collections for 'Business Insights'

Search Repository

search collection '<collection name or key>' for '<search string>' using ( [ page_size=<int> system_id=<system_id> edit_distance=<integer> display_first=<integer>]) show (data|docs) [ estimate only|return as data|save as '<csv_filename>' ]

Performs a document search of the Deep Search repository based on a given collection. The required USING clause specifies the collection to search. Use 'estimate only' to perform a general search, returning the potential number of hits.

Parameter:
- '' the name or index key for a collection. Use the command display all collections to identify collections.
- '' the search string for the search.

Search String Syntax: DeepSearch uses Elastic Search string query syntax, supporting operators like the following:
-- + signifies AND operation
-- | signifies OR operation
-- - negates a single token
-- \" wraps a number of tokens to signify a phrase for searching
-- * at the end of a term signifies a prefix query
-- ( and ) signify precedence
-- ~N after a word signifies edit distance (fuzziness)
-- ~N after a phrase signifies slop amount

USING clause Options:
- page_size= - result pagination, the default is None.
- system_id= - system cluster id, the default is the value 'default'.
- edit_distance= - Set the search word span criteria for key words for document searches. the default is 5. When set to 0, no snippets will be be returned.
- display_first= - If display_first > 0, the displayed result set will be truncated at the given number. The default is 0.

Clauses:
- show (data | docs ) - data Display structured data from within the documents or docs Display document context.
It is permitted to specify both in a single command e.g. show (data docs)
- estimate only - Determine the potential number of hits.
- return as data - For Notebook or API mode. Removes all styling from the Pandas DataFrame, ready for further processing.

Examples:
- Look for documents that contain discussions on power conversion efficiency:

search collection 'arxiv-abstract' for 'ide(\"power conversion efficiency\" OR PCE) AND organ*' using ( edit_distance=20 system_id=default) show (docs)

- Search the pubchem archive for 'Ibuprofen' and display related molecules' data:

search collection 'pubchem' for 'Ibuprofen' show (data)

- Search for patents which mention a specific smiles molecule:

search collection 'uspto-patent' for 'identifiers._name:\"smiles#ccc(coc(=o)cs)(c(=o)c(=o)cs)c(=o)c(=o)cs\"' show (data)
search for 'identifiers._name:"smiles#ccc(coc(=o)cs)(c(=o)c(=o)cs)c(=o)c(=o)cs" ' using (index_key='patent-uspto-smiles') show (data) save as 'myresults.csv'

Search For Patents Containing Molecule

This command searches for mentions of a specified molecules in registered patents.
As input parameters you can provide either a SMILES version of a molecule <smiles_molecule> or Inchi <inchi_molecule>, which can either be in key or string format.

If you use the SAVE AS clause, it will save a csv file to the current workspace. search for patents containing molecule'<Smiles-String>' [save as '<csv_file_name>']

Example Command: search for patents containing molecule 'CC(C)(c1ccccn1)C(CC(=O)O)Nc1nc(-c2c[nH]c3ncc(Cl)cc23)c(C#N)cc1F'

Search For Molecules in Patents

This command searches for molecules that are mentioned in the defined list of patents. If sourcing patents are from CSV or dataframe, these must contain a column with 'PATENT ID' or 'patent id' as the heading.
search for molecules in patents from list ['<Smiles-String>','<Smiles-String>'...] [save as '<csv_file_name>']

Example Command: search for molecules in patents from list ['CN108473493B','US20190023713A1']

Search For Similar Molecules

This command searches for molecules that are similar to the provided molecule or molecule substructure provided.

If you use the SAVE AS clause, it will save a csv file to the current workspace
search for molecules similar to '<Smiles-String>'

Example Command: search for similar molecules to 'C1(C(=C)C([O-])C1C)=O'
